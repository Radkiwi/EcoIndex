# Ecosystem Services Valuation -- New Zealand

This repository contains the code used to calculate **ecosystem service
valuations for New Zealand ecosystems** as part of the Eco‑index
methodology.

The valuation methodology follows the framework described in the
**Eco-index Product Deep Dive (pages 22--27)**. The model estimates the
economic value of ecosystem services using a **benefit transfer
approach**, where values derived from global ecological economics
literature are mapped to New Zealand ecosystem types.

The goal of this code is to provide a **transparent, reproducible
workflow** for estimating ecosystem service values spatially across
landscapes.

------------------------------------------------------------------------

# Purpose of the Model

The Ecosystem Services Valuation model estimates the **annual economic
value generated by ecosystems** through services such as:

-   Climate regulation
-   Water purification
-   Soil formation
-   Pollination
-   Flood mitigation
-   Habitat provision
-   Cultural and recreational value

These services are quantified using **per‑hectare ecosystem service
values**, which are applied to mapped ecosystem types.

The model outputs **spatial layers of ecosystem service value** which
can be used for:

-   ecological planning
-   restoration prioritisation
-   natural capital accounting
-   environmental impact assessments
-   biodiversity investment analysis

------------------------------------------------------------------------

# Valuation Methodology

The model uses a **benefit transfer approach**, a commonly used method
in ecological economics where previously estimated ecosystem service
values are transferred to new locations with similar ecosystem
characteristics.

The valuation coefficients used in the model are derived from major
ecosystem service valuation studies including:

-   Costanza et al. (1997; updated 2014)
-   de Groot et al. (2012)
-   Brander et al. (2012)
-   Ecosystem Service Value Database (ESVD)
-   Environmental Valuation Reference Inventory (EVRI)

These global ecosystem service values are **mapped to New Zealand
ecosystem types** using land cover and ecosystem classification
datasets.

# Required Input Data

To run the valuation model, users must assemble the required spatial
layers and lookup tables.

## Spatial Layers

Typical spatial inputs include:

  Dataset                      Purpose
  ---------------------------- ------------------------------------
  Land Cover Database (LCDB)   Identifies ecosystem types
  Wetlands datasets            Wetland ecosystem classification
  Forest / vegetation layers   Native forest and vegetation types
  Hydrology layers             Rivers and water bodies
  Protected area datasets      Ecological significance context

These layers are used to determine **which ecosystem type is present in
each spatial unit**.

## Lookup Tables

Lookup tables assign **ecosystem service values to ecosystem types**.

Typical lookup tables include:

  ------------------------------------------------------------------------------
  Table                              Purpose
  ---------------------------------- -------------------------------------------
  ecosystem_value_coefficients.csv   Monetary value per hectare per ecosystem
                                     type

  ecosystem_service_categories.csv   Mapping of services to ecosystem types
  ------------------------------------------------------------------------------

------------------------------------------------------------------------

# Preparing Input Data

Before running the valuation notebook:

1.  Ensure all spatial datasets use a common coordinate system
    (typically **NZTM2000 -- EPSG:2193**).
2.  Convert datasets to supported formats (GeoPackage or GeoTIFF).
3.  Ensure ecosystem classifications align with the lookup table
    ecosystem categories.
4.  Place datasets into the input data directory.

Example:

data/ input_layers/ landcover.gpkg wetlands.gpkg rivers.gpkg

    lookup_tables/
        ecosystem_value_coefficients.csv

------------------------------------------------------------------------

# Running the Valuation Model

Open the notebook:

Ecosystem_Valuations_NZ.ipynb

Steps:

1.  Update file paths to the input datasets.
2.  Load spatial ecosystem layers.
3.  Map ecosystem types to valuation coefficients.
4.  Calculate ecosystem service values per spatial unit.
5.  Generate valuation outputs.

# Outputs

The model generates spatial outputs describing ecosystem service value.

Typical outputs include:

  Output                    Description
  ------------------------- ------------------------------------------
  ecosystem_value_layer     Total ecosystem value per spatial unit
  service_specific_values   Values for individual ecosystem services
  summary_tables            Aggregated values for project areas

Outputs are typically stored as:

-   GeoPackage layers
-   GeoTIFF rasters
-   CSV summary tables

# Interpreting Ecosystem Service Values

Ecosystem service valuations represent **indicative estimates of
ecosystem contributions to human wellbeing**.

They should be interpreted as:

-   relative indicators of ecosystem importance
-   tools for comparing land management scenarios
-   inputs for environmental decision‑making

They are **not direct market prices** and should not be interpreted as
exact monetary values.

------------------------------------------------------------------------

# Limitations

Ecosystem service valuation models have several limitations:

-   Values are derived from **global studies rather than site‑specific
    measurements**
-   Ecosystem services may vary depending on local conditions
-   Some ecosystem services remain difficult to quantify economically
-   Cultural and intrinsic ecological values may not be fully
    represented

Despite these limitations, ecosystem service valuation provides a
**useful decision‑support tool** for integrating ecological value into
planning and investment.
